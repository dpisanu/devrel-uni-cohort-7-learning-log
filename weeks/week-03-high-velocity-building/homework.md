# Homework: Week 03

## Assignment title

Continue developing your project.

## Assignment summary

The Week 03 homework asked us to continue developing our project while thinking beyond the core build.

The main instruction was to consider the surrounding pieces a project may need, such as:

* onboarding
* tutorials
* examples
* community
* documentation
* landing page
* feedback systems
* distribution
* user education
* continued engagement

The assignment emphasized that not every supporting piece is relevant for every project. The goal was to think about the full experience around what is being built.

## Part 1: Think beyond the thing itself

Answer short questions about the project experience:

* How will people learn about the project?
* How will people learn how to use the project?
* How will the right people be attracted to try or engage with the project?
* What are potential ways to gather feedback from users or developers?
* What would make someone continue using or following the project over time?
* What other parts are still needed for the project?

## Part 2: Ship another iteration

Continue building the project and ship another meaningful iteration.

Examples included:

* publishing another content piece
* improving onboarding or documentation
* building additional functionality
* creating a walkthrough or tutorial
* improving UX or clarity
* creating a landing page
* running a small feedback session with users

The documentation questions for this part were:

* What did I build or improve this week?
* What feedback influenced the decision, and where did it come from?
* What part took the longest?
* Where did AI help accelerate the work?
* Where did I still rely on human judgment?

## Part 3: Build in public

Create a public progress update for LinkedIn, X, GitHub, or another platform.

The post should share:

* what improved or shipped this week
* a screenshot, draft, demo, or insight from the process

Optional additions:

* something learned
* a challenge encountered
* an open question

## My interpretation for this week

This homework became complicated because the original project reached a pivot point.

For the Open Source Readiness Field Guide, "continue developing the project" does not mean shipping more independent content. It means documenting the decision to stop building it as a standalone project, explain why that decision is responsible, and identify a better next move.

For the emerging PlainTrack idea, the homework becomes a way to shape the first project experience before building too much:

* Who is it for?
* How will they learn about it?
* How will they try it?
* How will I collect feedback?
* What would make the project useful enough to return to?
