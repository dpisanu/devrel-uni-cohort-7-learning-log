# Week 01: Building & DevRel in the Age of AI

## Overview

**Week:** Week 01  
**Theme:** Building & DevRel in the Age of AI  
**Project framing:** Open Source Readiness Field Guide  
**Status:** Draft

Week 01 focused on grounding the DevRel University Cohort 7 journey, understanding DevRel as an intentional practice, and framing a concrete project direction in the context of AI-shaped learning and building.

## My Week 01 outcome

My project direction became:

**Open Source Readiness Field Guide**

A practical orientation resource for small and midsize companies that want to better understand open source usage, responsibility, compliance, sustainability, software supply chain topics, and good open source citizenship.

This is not intended to be legal advice, a compliance framework, or a replacement for an Open Source Program Office. It is a learning and enablement artifact that can help companies ask better questions and take a more intentional first step.

For the project framing and weekly retrospective, see [`project-brief.md`](./project-brief.md).

## Key takeaways

- DevRel needs direction, stakeholder alignment, feedback loops, and sustained action.
- AI changes how quickly we can draft, build, and learn, but it does not remove the need for judgment, context, and trust.
- A broad interest in open source and OSPO work became a smaller project artifact that I can explain, share, and iterate on.
- The first useful step is not to build a complete framework. It is to create an approachable entry point.
