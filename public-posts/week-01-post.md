I’m catching up on documenting my DevRel University **Cohort 7** journey in public.

This is my retrospective note for **Week 01** where we focused on understanding DevRel more intentionally:
* What is DevRel?
* What does success look like?
* Which objectives does it serve?
* Which practices show up as recurring patterns that we can adapt?

To make this tangible, we were asked to bring in a project we had wanted to work on for a long time and use it as a reflective work object.

For me, that project is connected to **open source readiness for small and midsize companies**.

Just before starting #DevRelUni, I attended a Linux Foundation event on open source in industry and the event reinforced a topic I had already been thinking about: Many SMEs already depend deeply on open source, but not all of them have dedicated OSPO structures, formal strategies, or internal communities to support responsible usage.

So my working project became: **Open Source Readiness Field Guide**

A practical starting point for SMEs that want to better understand open source usage, software supply chain, sustainability, contribution, and good open source citizenship.

I documented the Week 01 exercise in my GitHub repository.

So if you are interested in following along with this thought experiment, head over here: https://github.com/dpisanu/devrel-uni-cohort-7-learning-log

Stay tuned for Week 02.
I already know what happens next. 😁
